gulp的理解

  • 概念理解：gulp跟js没有关系，跟编写程序的功能也没有关系。跟做项目有关系，是工程化（把项目叫作工程）有关的一个工具，这个工具能干什么？

  • 作用：

    （1）能够完成代码的压缩

    （2）合并，复制，监听，ES6的代码转换，启动服务器等等。

Gulp的使用步骤：

1. 搭建操作系统的环境：

（1）安装nodeJS（gulp是基于nodeJS的，所以要安装nodeJS）

​ 测试：在命令行输入：node -v ，能够显示版本号(version)，就说明没有问题， 如果提示:不是内部或者外部命令，则配置环境变量

（2）测试npm：在命令行输入：npm -v

​ 注：Npm(nodeJS Package Manager)：nodeJS的包管理工具

​ 安装node的另外一个目的，是为了使用npm。npm在安装node时，已经同时安装过了

​ 我们在开发项目时，会使用第三方的js代码，比如：jQuery等等。如果一个项目中使用的第三方js代码非常多，意味着，我们需要在不同的网站下载不同的js代码（js库），非常麻烦，而且，如果js代码之间互相有依赖，那么还需要把依赖的js下载下来。非常麻烦。

​ Npm把所有第三方js库收集在一起，放在npm的服务器上，这样，程序员每次下载时，统一从npm下载。而且，npm还可以管理js之间的依赖关系。

​ 一个包就是一个js库。所以，nodeJS Package Manager就是nodeJS专门管理js库的一个工具。

（3）安装全局的gulp：命令行输入：npm install gulp -g

全局安装gulp的意思是：给整个操作系统安装gulp。跟安装qq是同样的道理，只不过，平时安装qq是用可视化的方式，而安装gulp用的是命令行的方式

Install：安装的意思，可以简写为i。

-g:表示global，全局。

​ 测试：在命令行输入:gulp -v

2. 搭建项目环境（本地安装gulp）：

（1）建立项目文件夹：

从盘符到项目文件夹中间，任何一级都不要有中文，项目名也不要用中文，项目名不要用“gulp”。

如： d:\我的\prj 这不行；

​ d:\my\项目 这也不行；

​ d:\my\gulp 这也不行

（4）本地安装gulp

​ 即，在项目目录下安装gulp。

​ 在命令行里（把命令行的当前目录切换到项目目录下），

​ 运行：npm install gulp --save-dev

3. 在项目中使用gulp：

（1）在项目目录下建立文件 “gulpfile.js”（此文件名不能乱写），记住：这个文件一定是放在项目根目录下。不要放在项目的其它子文件里，文件名就是“gulpfile.js”，不要乱起。

（2）在gulpfile.js里实现功能：

① 复制文件：

const gulp = require(“gulp”);

// require(“gulp”); 相当于在html的

// require(“gulp”); 是用js文件引入的js文件，这是模块化的做法。

//写一个复制文件的任务

gulp.task(“copy-html”,async ()=>{

​ //以下表示，把当前目录下的index.html文件拷贝至当前目录下的dist目录下。

​ gulp.src(“index.html”)

​ .pipe(gulp.dest(“dist”));

});

② 增加监听文件：（随时监视某个文件是否改变，如果改变则执行相应的任务）

使用watch：gulp.task(“watchall”, async function() {

// 监听Index.html文件的变化。如果index.html文件变化了，则执行后面的回调函数

gulp.watch(“”, );

})

③ 压缩js文件：

Gulp本身只能完成复制文件，监听文件等简单功能，如果要完成更多的功能，gulp提供很多接口，可以使用插件

压缩js文件，就需要 gulp-uglify 插件。

安装 gulp-uglify 插件： npm install gulp-uglify –save-dev

④ 合并js

使用gulp-concat 插件，把若干个小js文件合并成一个大的js文件，可以减少http的请求（提高项目的性能）。

安装： npm install gulp-concat –save-dev

⑤ 重命名

使用gulp-rename 插件

安装： npm install gulp-rename –save-dev

⑥ 压缩css

Gulp-minify-css

安装： npm install gulp-minify-css –save-dev
